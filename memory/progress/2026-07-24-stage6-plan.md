# Stage 6 plan

After Stage 5 closed (honest ATE metric + real loop closure, both
verified), the user asked for an HTML explainer of why the ATE magnitude
still looks terrible (published artifact: "Why the ATE numbers are so
bad," four layers — a base tuning gap, real uncorrected drift, an
unexplained scale anomaly, and the now-fixed measurement-honesty issue).
Immediately followed by: "write plan/STAGE6.md for fixing these four
layers." Wrote the plan directly off that same diagnostic work — no new
investigation needed, since Stage 5's own decisions (`0020`, `0021`) and
`docs/RESULTS.md` already had every number.

## The three goals (Layer 4 is already fixed, doesn't need its own goal)

1. **Close the base tuning gap with real structural work** — analytic
   IMU Jacobians (`crates/slam-optim/src/imu_factor.rs` still uses
   numerical/central-difference ones, confirmed by reading the code
   fresh rather than trusting memory of `decisions/0006`; reprojection's
   own Jacobians are already analytic, a real asymmetry worth noting)
   plus real preintegration covariance propagation (replacing
   `solver_config_from_sensor_noise`'s simplified formula, already
   measured and reverted in `decisions/0016`).
2. **Remove loop closure's own correction ceiling** — a real sparse
   pose-graph solver, replacing `optimize_pose_graph`'s dense O(n^3) LU
   solve (the exact thing that forced Stage 5 M2's stride-4 workaround
   and its own measured RPE cost).
3. **Root-cause the scale-consistency anomaly** — `decisions/0020`
   ruled out two causes (calibration, disproportionate recovery-step
   size) but never tested its own leading hypothesis (IMU-vs-vision
   residual weighting inside the windowed optimizer). This goal
   actually tests it, with a real ablation, instead of leaving it as an
   unconfirmed guess.

## Ordering rationale

Goals are ordered by tractability, not by importance — Goal 3 (the most
foundational, since a real scale bug would affect everything downstream)
is also the most open-ended (no confirmed lead yet, `decisions/0020`'s
own hypothesis untested), so it goes last, matching `plan/STAGE5.md`
M0's own precedent for how an investigation that might not converge
should be scoped: a hard "document what's ruled in/out and stop" branch
(M7) rather than forcing a fix that isn't backed by real causation.
Goals 1 and 2 are both concretely scoped, already-identified pieces of
work (not new discoveries) — safer to front-load.

## Milestone structure

8 milestones (M0-M7), grouped in blocks per goal: M0-M2 (Goal 1,
analytic Jacobians then covariance propagation), M3-M4 (Goal 2, sparse
solver then using the resulting headroom), M5-M7 (Goal 3, instrument
drift then test the weighting hypothesis then fix-or-document). Each
accuracy milestone commits to re-measuring on all 5 sequences with both
ATE metrics (`plan/STAGE5.md` M1's prefix-aligned one, not just the
legacy whole-trajectory one) — carrying that stage's own honesty
discipline forward rather than letting it lapse now that a new stage
has started.

## Real risks named up front, not discovered later

- Any of the three goals could turn out to be its own stage — `docs/
  RESULTS.md` already called the covariance work "a real, larger,
  separate undertaking" before this stage existed; explicit permission
  to re-scope mid-milestone if effort estimates blow up, rather than
  quietly under-delivering on all three to preserve one finished-looking
  stage.
- A wrong analytic Jacobian is a silent-bug risk (still "converges," to
  a worse answer) — `plan/STAGE1.md`'s own Risks section named this
  exact danger for the optimizer originally; M1's finite-difference
  verification against real (not just synthetic) data is the guard.
- Every milestone touching solver cost (M1-M4 especially) must re-verify
  the real-time bar (`whole_run_factor() <= 1.0`, `plan/STAGE4.md` M1)
  directly — this project has regressed that bar once already this
  session (Stage 5 M2's own dense-pose-graph mistake) by not applying an
  already-learned lesson to a new call site; named explicitly so it
  doesn't happen a third time.
